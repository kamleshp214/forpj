JavaScript Fundamentals (The Foundation)

Must Master First

Explain Hoisting in JavaScript.

Answer: Hoisting is JavaScript's default behavior of moving declarations (var, function) to the top of their scope before code execution. let/const are also hoisted but not initialized (Temporal Dead Zone).

Why Interviewers Ask: Shows understanding of execution context and variable environment.

console.log(x); // undefined (not error for var)
var x = 5;

// vs
console.log(y); // ReferenceError
let y = 10;

What is Closure? Real-world use case.

Answer: A closure is a function that remembers its outer variables even after the outer function has returned. Used in data privacy, module pattern, currying.

Why Asked: Core concept for React hooks, event handlers, and Node.js callbacks.

function makeCounter() {
  let count = 0;
  return function() {
    return ++count;
  };
}
const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2

Event Loop in detail with examples.

Answer: Call Stack → Microtask Queue (Promises) → Macrotask Queue (setTimeout, I/O). Microtasks execute before next macrotask.

Why Asked: Critical for understanding async behavior in Node.js and React.

console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1 4 3 2

this keyword - 4 binding rules.

Answer: 1. Default (global/window), 2. Implicit (object calling), 3. Explicit (call/apply/bind), 4. New (constructor). Arrow functions lexically bind this.

Why Asked: Essential for React class components and event handlers.

Prototype Chain and Inheritance.

Answer: Every object has __proto__ linking to its constructor's prototype. Used for method sharing before ES6 classes.

Why Asked: Understanding under-the-hood of classes and React component inheritance.

Async/Await vs Promises vs Callbacks.

Answer: Callbacks → Promise → async/await (syntactic sugar). async/await makes error handling easier with try/catch.

Why Asked: You'll write async code daily in Node.js APIs.

MongoDB Mastery (NoSQL Database)

Intermediate to Advanced

Embedding vs Referencing - When to use which?

Answer: Embedding: For contained data (one-to-few), faster reads, atomic updates. Referencing: For large/many-to-many, avoids duplication, flexible.

Why Asked: Core data modeling decision in MERN apps.

Pro Tip: Embed comments in blog post (few), reference products in orders (many).

Explain Aggregation Pipeline stages with example.

Answer: $match (filter), $group (aggregate), $project (reshape), $sort, $limit, $lookup (join), $unwind (de-array).

// Top 5 highest paying job categories
db.jobs.aggregate([
  { $match: { salary: { $gt: 50000 } } },
  { $group: { _id: "$category", avgSalary: { $avg: "$salary" } } },
  { $sort: { avgSalary: -1 } },
  { $limit: 5 }
])

Indexes: Types, Compound, Partial, TTL.

Answer: Single field, compound (order matters), text (search), geospatial, TTL (auto-expire).

Pro Tip: Always use explain('executionStats') to verify index usage.

Transactions in MongoDB - When possible?

Answer: Since 4.0, multi-document ACID transactions in replica sets. Use for critical operations like bank transfers.

Schema Validation in MongoDB.

Answer: Define JSON schema in collection creation for required fields, types, enums, custom validators.

Why Asked: Prevents bad data before Mongoose validation.

Change Streams - Real-time updates.

Answer: Watch collection for changes (insert/update/delete). Used for real-time features like notifications.

Example Use: In HireSphere, notify when new job matches user skills.

Middleware lifecycle and error handling.

Answer: Middleware chain → next() → next middleware. Error middleware (4 params) catches errors.

Authentication: JWT vs Sessions vs OAuth.

Answer: JWT: Stateless, scalable. Sessions: Server-side (Redis). OAuth: Third-party login.

Rate Limiting strategies.

Answer: express-rate-limit, Redis token bucket. Prevent brute force/DDoS.

Streams for large files - Why and How?

Answer: Prevents memory overflow. Use fs.createReadStream() for uploads/downloads.

Use Case: Resume Builder PDF streaming.

Clustering and PM2.

Answer: Cluster forks workers. PM2 manages clusters, zero-downtime reloads.

Security Best Practices (Top 10).

Answer: Helmet, CORS, input validation (Joi), rate limiting, HTTPS, no-eval, dependency scanning, env variables.

Must-Know Code Patterns & Snippets

Protected Route Middleware (Express)

const protect = async (req, res, next) => {
  try {
    const token = req.cookies.token || req.header('Authorization')?.replace('Bearer ', '');
    if (!token) return res.status(401).json({ error: 'Unauthorized' });

    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = await User.findById(decoded.id).select('-password');
    next();
  } catch (err) {
    res.status(401).json({ error: 'Token invalid' });
  }
};

Why this matters:

Checks both cookie and header, securely populates req.user for protected routes.

Custom Hook: useForm with Validation

const useForm = (initialValues, validate) => {
  const [values, setValues] = useState(initialValues);
  const [errors, setErrors] = useState({});
  const [touched, setTouched] = useState({});

  const handleChange = (e) => {
    const { name, value } = e.target;
    setValues({ ...values, [name]: value });
  };

  const handleBlur = (e) => {
    const { name } = e.target;
    setTouched({ ...touched, [name]: true });
    const validationErrors = validate(values);
    setErrors(validationErrors);
  };

  return { values, errors, touched, handleChange, handleBlur };
};

Why this matters:

Reusable form logic with validation on blur — perfect for login/signup forms.

MongoDB Transaction Example

const session = await mongoose.startSession();
session.startTransaction();
try {
  await Order.create([order], { session });
  await Product.updateOne(
    { _id: productId },
    { $inc: { stock: -quantity } },
    { session }
  );
  await session.commitTransaction();
} catch (error) {
  await session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}

Why this matters:

Atomic inventory update with order creation — critical for e-commerce.
